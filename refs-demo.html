=> Refs are to access DOM nodes or React elements created in the render method.

    => When you change a ref, it doesn't cause the component to re-render.
      This is a key difference from changing state or props.
      When state or props change, React will typically trigger a re-render of the component.
      However, changing a ref won't cause this

    => Ref allows a React component to hold a reference to a DOM element or another React component.
      This reference can be used to interact with or access the properties of that element or
      component at a later time, even after it has been rendered.
